MQTT Testing & Monitoring

Step-by-Step AWS IoT Core & CloudWatch Guide

1
Create IoT Thing (Device)

Set up your first IoT device in AWS IoT Core

Steps:

  1. Login to AWS Console → IoT Core
  2. Navigate to Manage → Things → Create
  3. Choose Create a single thing
  4. Name: TestMQTTDevice
  5. Skip Type and Group → Click Next
This creates the virtual representation of your physical device in AWS IoT.
2
Generate Certificates & Policy

Secure your device with certificates and access policies

Certificate Setup:

  • Choose Auto-generate new certificates
  • Download all three files:
    • device.pem.crt (Certificate)
    • private.pem.key (Private Key)
    • AmazonRootCA1.pem (Root CA)
  • Click Activate

Policy Creation:

MQTT Policy 
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "iot:*",
      "Resource": "*"
    }
  ]
}

Name: MQTTTestPolicy → Create and attach to certificate

3
Test MQTT Messaging

Send and receive MQTT messages using AWS IoT Test Client

Testing Steps:

  1. Go to Test → MQTT Test Client
  2. Subscribe to topic: test/topic
  3. Publish a message:
    • Topic: test/topic
    • Payload: {"message": "Hello from AWS MQTT"}
  4. Confirm message appears in subscription window
This validates your MQTT connection and certificate setup.
4
Enable CloudWatch Logs

Configure logging for monitoring and debugging

Logging Setup:

  1. Go to IoT Core → Settings
  2. Scroll to CloudWatch Logs
  3. Enable logging with:
    • Log group: /aws/iot/MQTTTestLogs
    • Log level: DEBUG
  4. Save settings
5
View CloudWatch Logs

Monitor and analyze your MQTT traffic logs

Log Analysis:

  1. Go to CloudWatch → Logs → Log groups
  2. Open /aws/iot/MQTTTestLogs
  3. Use filters like:
    Log Filter Examples 
    test/topic
clientId
errorCode
disconnect
6
Create Metrics Dashboard

Build a comprehensive monitoring dashboard

Dashboard Creation:

  1. Go to CloudWatch → Metrics
  2. Choose namespace: AWS/IoT
  3. Select key metrics:
    • NumberOfMessagesPublished
    • NumberOfMessagesReceived
    • NumberOfMessagesDropped
  4. Go to Dashboards → Create dashboard
  5. Name: MQTTMonitoringDashboard
  6. Add widgets for real-time monitoring
7
Python MQTT Simulation

Automate testing with Python scripts

Python Setup:

Install Dependencies 
pip install paho-mqtt

Python Script:

MQTT Test Script 
import paho.mqtt.client as mqtt
import json

# Configure client
client = mqtt.Client()
client.tls_set(ca_certs="AmazonRootCA1.pem",
               certfile="device.pem.crt",
               keyfile="private.pem.key")

# Connect to AWS IoT
client.connect("YOUR_ENDPOINT.iot.YOUR_REGION.amazonaws.com", 8883)

# Publish test message
message = {"message": "Hello from Python", "sensor_data": 25.5}
client.publish("test/topic", json.dumps(message))

print("Message published successfully!")
client.loop_forever()

Replace: YOUR_ENDPOINT with your IoT Core endpoint from Settings → Endpoint

8
Log Analysis & Alarms

Set up proactive monitoring and alerts

Log Insights Query:

CloudWatch Logs Query 
fields @timestamp, @message
| filter @message like /test\/topic/
| sort @timestamp desc
| limit 20

Alarm Setup:

  1. Go to CloudWatch → Alarms → Create Alarm
  2. Choose metric: NumberOfMessagesDropped
  3. Set threshold: > 5 messages in 5 minutes
  4. Configure SNS for email/SMS alerts
